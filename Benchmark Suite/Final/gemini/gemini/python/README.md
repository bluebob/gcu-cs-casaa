# Python Vulnerability Benchmark Set

This document details a set of 12 vulnerabilities in Python and its ecosystem, generated by Gemini for a degree research project.

## Vulnerabilities

*   **CVE-2025-56005** - PLY (Python Lex-Yacc) RCE
    *   **Impact:** 9.8 (Critical) - Unsafe pickle loading in the `yacc()` function allows for remote code execution.
*   **CVE-2020-27619** - CPython eval()
    *   **Impact:** 9.8 (Critical) - The `multibytecodec_support.py` test calls `eval()` on content retrieved via HTTP, allowing code execution if the connection is intercepted.
*   **CVE-2025-43859** - h11 Request Smuggling
    *   **Impact:** 9.1 (Critical) - The `h11` HTTP/1.1 library accepts malformed Chunked-Encoding bodies, potentially bypassing reverse proxy controls.
*   **CVE-2024-25128** - Flask-AppBuilder Auth Bypass
    *   **Impact:** 9.1 (Critical) - Incorrect authentication when using OpenID, allowing attackers to forge requests and gain privileged access.
*   **CVE-2015-20107** - CPython mailcap Injection
    *   **Impact:** 7.6 (High) - The `mailcap` module fails to escape shell commands, allowing command injection via untrusted input.
*   **CVE-2024-6232** - CPython tarfile ReDoS
    *   **Impact:** 7.5 (High) - Regular expression denial-of-service (ReDoS) via specifically crafted tar archives during header parsing.
*   **CVE-2024-24762** - python-multipart ReDoS
    *   **Impact:** 7.5 (High) - Affects FastAPI and Starlette; inefficient RegEx in `Content-Type` parsing allows CPU exhaustion.
*   **CVE-2024-47081** - Requests Credential Leak
    *   **Impact:** 7.5 (High) - Maliciously crafted URLs can cause the `Requests` library to leak `.netrc` credentials to third parties.
*   **CVE-2019-20907** - CPython tarfile DoS
    *   **Impact:** 7.5 (High) - An infinite loop in `tarfile.py` when processing certain PAX headers in TAR archives.
*   **CVE-2020-26116** - CPython CRLF Injection
    *   **Impact:** 7.2 (High) - `http.client` allows CRLF injection via the method parameter, potentially leading to request splitting.
*   **CVE-2026-30922** - Uncontrolled Recursion in `pyasn1`
    *   **Impact:** 7.5 (High) - Uncontrolled recursion in the BER decoder allows for a denial of service.
*   **CVE-2024-12254** - Use-After-Free in Decompression Modules
    *   **Impact:** 8.1 (High) - A Use-After-Free (UAF) vulnerability exists in `lzma.LZMADecompressor`, `bz2.BZ2Decompressor`, and `gzip.GzipFile`.
