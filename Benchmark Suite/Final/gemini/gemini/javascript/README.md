# JavaScript Vulnerability Benchmark Set

This document details a set of 9 vulnerabilities in JavaScript and its ecosystem, generated by Gemini for a degree research project.

## Vulnerabilities

*   **CVE-2024-21896** - Node.js Permission Model Bypass
    *   **Impact:** 9.1 (Critical) - A flaw in the Node.js Permission Model allows attackers to bypass `--allow-fs-read` and `--allow-fs-write` restrictions.
*   **CVE-2022-24999** - `qs` prototype pollution
    *   **Impact:** 7.5 (High) - A prototype pollution vulnerability in the `qs` library that can lead to a denial of service.
*   **CVE-2017-16138** - `mime` ReDoS
    *   **Impact:** 7.5 (High) - A Regular Expression Denial of Service (ReDoS) vulnerability in the `mime` npm package.
*   **CVE-2014-6394** - `serve-static` directory traversal
    *   **Impact:** 7.5 (High) - A directory traversal vulnerability in the `serve-static` module.
*   **CVE-2022-31147** - jQuery Validation Plugin ReDoS
    *   **Impact:** 7.5 (High) - A Regular Expression Denial of Service (ReDoS) vulnerability in the `url2` method of the jQuery Validation Plugin.
*   **CVE-2021-23632** - `git` (npm package) RCE
    *   **Impact:** 6.6 (Medium) - Remote Code Execution vulnerability within the Git npm package.
*   **CVE-2022-25881** - `http-cache-semantics` ReDoS
    *   **Impact:** 7.5 (High) - A Regular Expression Denial of Service (ReDoS) vulnerability in the `http-cache-semantics` package.
*   **CVE-2024-43796** - Express open redirect
    *   **Impact:** 7.4 (High) - An open redirect vulnerability exists in Express when using certain configurations of `res.redirect()`.
*   **CVE-2024-43800** - Express ReDoS
    *   **Impact:** 7.5 (High) - A Regular Expression Denial of Service (ReDoS) vulnerability in the way Express handles certain headers or path patterns.
